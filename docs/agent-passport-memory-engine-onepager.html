agent-passport + 记忆稳态引擎

把 Agent 的“身份证、档案室、工作记忆、保险柜、执行门禁、恢复预案”合成一个本地优先的长期运行底座。

不是新大模型 它让现有 Agent 更稳定、更可恢复、更可管住，而不是替代底层模型。

一句话说明

合并后，agent-passport 不只是 Agent 的身份护照，也不只是记忆库，而是一套本地 Agent Runtime：外层管身份、权限、安全和恢复，内层管长期记忆、上下文重建、来源可信度和记忆整理。

解决的真实问题

  • Agent 换窗口、断线、重启后容易像“换了人”。
  • 长聊天历史不是长期记忆，越塞越乱、越贵、越容易漏重点。
  • 模型容易把“事实、传闻、推断、摘要”混成一团。
  • 自动执行如果没有风险分级和沙箱，很容易越权或误操作。
  • 本地数据、恢复包、迁机流程如果不演练，出事时接不上。

合并后的工作方式：先稳住主体，再稳住记忆

1. 固定 Agent 身份 resident agent（常驻 Agent）：本机默认认定的同一个 Agent，避免每个聊天窗口都重新猜“我是谁”。
2. 分层保存记忆 working / episodic / semantic / identity memory（工作、事件、规律、身份记忆）：把短期任务、历史事件、长期规律和身份事实分开存。
3. 标记来源可信度 source monitoring（来源监测）：区分已验证、亲眼观察、别人报告、系统推断，防止把猜测当事实。
4. 自动整理记忆 salience / consolidation（显著性 / 巩固）：重要且可靠的内容优先保留，低价值噪声逐步降权。

运行链路：每轮任务都走同一套可检查流程

5. 装配上下文 context builder（上下文装配器）：开工前先把身份、任务、证据、记忆和恢复点整理给模型。
6. 生成后再校验 verifier（回复校验器）：检查 Agent ID、DID、钱包、授权、证据引用，降低关键事实漂移。
7. 执行先过门禁 risk tier + sandbox（风险分级 + 沙箱）：低风险快走，高风险确认，命令放进受限环境执行。
8. 失败可恢复 checkpoint / compact boundary（检查点 / 恢复边界）：断线、异常退出后，从最近可靠存档接着跑。

当前可验证能力

  • 本地身份、DID、授权、事件哈希链记录。
  • 五层记忆视图：感知、工作、情节、语义、身份。
  • 来源监测、显著性、可信度、记忆巩固和自适应遗忘。
  • 上下文装配、回复校验、记忆压缩、恢复边界。
  • 管理令牌、短时 read session、字段脱敏、受限执行审计。
  • 加密本地存储、恢复包、恢复演练、初始化包。

真实边界，不夸大

  • 不是新大模型，也不承诺让模型永远不幻觉。
  • 不是神经科学意义上的人脑复刻，而是工程化记忆稳态。
  • 不是全球身份协议已完成，当前优先做单机单 Agent 跑稳。
  • 不是链上不可篡改系统，而是本地加密、哈希链和可回放审计。

对客户的价值

普通 Agent 像临时聊天员工，记忆靠窗口、执行靠提示、出错靠重问。合并后的 agent-passport 更像一个可长期工作的本地数字员工：有身份、有档案、有记忆整理、有执行门禁、有恢复预案、有审计记录。它优先解决“稳定、安全、可恢复、可追责”，再谈更大的多设备和协议化。
