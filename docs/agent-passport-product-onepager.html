agent-passport

给 AI Agent 的本地运行护照：让一个 Agent 在本机拥有稳定身份、长期记忆、权限边界、恢复流程和审计记录。

不是新大模型 它是 Agent Runtime 底座：管身份、记忆、恢复、安全和执行边界。

一句话说明

agent-passport 当前阶段是“单机单 Agent、本地优先、可恢复、可审计的 Agent Runtime”。普通理解：它不是替你训练一个新模型，而是给现有 Agent 配一套本地身份证、工作记忆、保险箱、操作门禁和事故记录本。

要解决的真实痛点

  • 聊天窗口一关，Agent 就像换了人，身份和任务很难连续。
  • 长聊天历史不是长期记忆，越塞越贵、越乱、越容易漏重点。
  • Agent 会漂移、会编造、会把身份、证据、权限说错。
  • 自动执行如果没有边界，误操作、越权和追责都会变成风险。
  • 本地数据、密钥、恢复包如果没有正式流程，重装或迁机时容易断档。

它怎么解决：从“聊天助手”变成“可运行主体”

1. 先固定身份 resident agent（常驻 Agent）：这台机器默认认定的同一个 Agent。它让系统不再每次靠聊天上下文猜“我是谁”。
2. 再沉淀记忆 ledger / minutes / snapshot / evidence（账本、纪要、任务快照、证据）：把该长期保存的信息放到线程外，不靠无限聊天记录硬撑。
3. 回答前先装上下文 context builder（上下文装配器）：开工前先把相关身份、任务、证据和恢复点拿齐，让模型少凭空发挥。
4. 回答后再校验 verifier（回复校验器）：检查 Agent ID、DID、钱包、授权、证据引用等硬字段，防止把关键事实说错。

安全与恢复路径

5. 执行先过门禁 risk tier（风险分级）：低风险快走，高风险先确认，critical 动作进入更严格的授权路径。
6. 命令进沙箱 sandbox broker（受限执行代理）：把命令放进隔离子进程，限制能读哪里、写哪里、连哪里。
7. 数据加密落盘 encrypted envelope（加密信封）：本地账本写入磁盘前先加密；密钥优先使用系统保护层，文件只做回退。
8. 恢复必须演练 recovery bundle / rehearsal（恢复包 / 恢复演练）：不只生成备份，还要真试一次能不能恢复。

当前可验证能力

  • 本地身份、DID、授权、事件哈希链记录。
  • 本地优先记忆检索、上下文装配、回复校验。
  • 管理令牌、短时 read session、资源绑定和字段级脱敏。
  • 加密本地存储、恢复包、恢复演练、初始化包。
  • 受限执行层、自动恢复/续跑审计、smoke/guard/soak 验证链。

真实边界，不夸大

  • 当前不是全球唯一身份网络，而是本机命名空间里的稳定身份。
  • 当前不是新大模型，也不承诺让模型永远不幻觉。
  • 当前不是链上不可篡改账本，而是本地加密、哈希链和可回放审计。
  • 当前优先做单机单 Agent 跑稳，再扩展多设备和协议化。

对客户的价值

如果说普通 Agent 像一个只能活在聊天窗口里的临时助手，agent-passport 要做的是让它变成一个可以长期住在本机、能记事、能恢复、被限制、可追责的工作伙伴。它优先解决“能不能稳定、能不能安全、能不能恢复、能不能审计”，再谈更大的平台化和协议化。
