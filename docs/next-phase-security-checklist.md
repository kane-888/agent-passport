# agent-passport 下一阶段实施清单

这份清单讨论的是 `agent-passport` 下一阶段怎么从“能跑”收成“能交付”；底层安全、恢复与执行边界仍由 `agent-passport 记忆稳态引擎` 提供。

## 目标

下一阶段的目标不是继续扩概念，而是把已经落地的 3 条基线收成真正可交付的闭环：

1. 安全架构
2. 本地存储加密与正式恢复
3. 受限执行层

判断标准只有 3 个：

- 出问题时能不能先保全现场
- 机器坏了时能不能按固定顺序救回来
- 模型理解错了时能不能被边界拦住，而不是直接碰宿主系统

## 当前判断

当前分支已经不再是“从 0 到 1”：

- 已有安全姿态、信任边界、事故姿态切换和只读/禁执行/panic 门禁
- 已有账本加密、Keychain 接入、恢复包、恢复演练、初始化包和正式 runbook
- 已有 broker / worker 隔离、系统级 broker sandbox、能力 allowlist、预算和审计记录
- 已有 operator 安全手册与跨机器恢复演练固定流程文档
- 已有恢复周期 / 轮换触发 / 清理自动化边界的运行态状态与制度文档
- 公开运行态 `/` 已收口成健康、正式恢复周期、自动恢复边界、可用入口 4 个只读答案；下一层页面不再混在首页
- `/repair-hub` 已收口成受保护修复中枢，`/api/offline-chat/thread-startup-context?phase=phase_1` 继续承载线程真值与“先串行收口、再最小必要并行”的协作口径；首页 `/` 不再消费这两类上下文

这意味着：

- 可以继续内部 Alpha
- 还不适合把“恢复”和“受限执行”对外讲成已经彻底封顶
- 下一步重点应该从“有没有”切到“够不够稳、能不能交付、能不能长期维护”

## 1. 安全架构

### 当前已落地

- 安全姿态从 normal 到 `read_only` / `disable_exec` / `panic` 已有明确控制面
- 信任边界已经拆成写接口、敏感读、系统保护层、正式恢复、自动恢复、受限执行
- 高风险动作不会默认直达执行层

### 还差什么

- 把 operator 手册和周期演练接到真实值班制度或定时提醒
- 把 key rotation、break-glass、证据保全补成标准动作

### 达标标准

- 任一高风险异常都能回答“先锁哪里、留什么证据、谁来继续”
- 安全姿态切换后，写入/执行/恢复门禁行为一致
- 公开运行态 `/`、接口、文档对同一件事给出同一结论

## 2. 本地存储加密与正式恢复

### 当前已落地

- 账本默认以加密 envelope 落盘
- store key / signing key 已有系统保护层接入
- 恢复流程已收成固定顺序：
  1. 保护密钥与账本
  2. 导出恢复包
  3. 执行恢复演练
  4. 导出初始化包
- `/` 已能给出正式恢复周期与自动恢复边界概览，`/api/device/setup` 已能给出正式 runbook、最近证据和下一步

### 还差什么

- 把跨机器恢复演练和恢复周期状态接到真实定时执行或值班提醒
- 把恢复包口令保管、轮换和失效策略写成制度
- 把恢复包、演练、初始化包的保留/轮换从手动触发磨成固定执行

### 达标标准

- 新机器接管时不需要靠人工拼装 JSON 或临时猜路径
- 任意一次密钥轮换后，都能明确知道要重跑哪几步
- 正式恢复没达标时，自动恢复不会被误当成“已经备份完成”

## 3. 受限执行层

### 当前已落地

- broker / worker 双层隔离已经存在
- broker 在可用环境下优先落到系统级 sandbox
- 文件、网络、命令、输入输出、超时都有预算与 allowlist
- 最近一次受限执行结果与隔离审计可以回看

### 还差什么

- 把风险等级到能力放行的映射再收紧一层
- 把高风险确认钩子和 UI 提示做得更一致
- 把策略退化、allowlist 误放行、预算越界补成更硬的回归测试

### 达标标准

- 模型即使理解错了，也很难直接越权碰到宿主系统
- 每次执行前都能回答“为什么能放、为什么被挡、挡住后谁来接力”
- 受限执行层退化时，`/api/security` 和值班入口会第一时间暴露风险，而不是静默继续

## 实施顺序

按最短路径，只做这 3 件事：

1. 先把安全姿态与操作责任表补齐
2. 再把正式恢复从“能跑”磨到“能交接”
3. 最后把受限执行的策略映射和确认钩子收紧

不要反过来。因为：

- 没有安全责任表，恢复和执行一旦出事就没人知道先停哪
- 没有正式恢复闭环，自动续跑再顺也只是临时接力
- 没有受限执行硬边界，前两条再完整也挡不住错误操作放大

## 发布门槛

### 继续内部 Alpha

- 当前状态已经满足，可以继续

### 进入私测 Beta 前必须补齐

- 操作级安全责任表
- 可重复的跨机器恢复演练
- 更硬的受限执行退化回归

### 对外 Preview 前必须补齐

- 口令与密钥轮换制度
- 恢复包/初始化包保留与轮换策略
- 高风险动作回滚与事故复盘模板

## 一句话收口

下一阶段不是再证明“我们有这些能力”，而是把它们磨到真正能保护、能恢复、能交接。
